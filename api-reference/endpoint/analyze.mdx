The Analyze endpoint is the core of StackSee's API. It allows you to analyze any website to extract technology stack information, including frameworks, libraries, hosting providers, and branding details.

## Features

- **Technology Detection**: Automatically identifies frameworks, libraries, CMS platforms, and other technologies
- **Brand Extraction**: Captures logos, colors, and other branding elements
- **Deep Scanning**: Optionally crawl multiple pages for more comprehensive analysis
- **Smart Caching**: Configurable caching with freshness controls to balance speed and accuracy
- **Async Processing**: Support for webhook notifications for long-running analyses

## Use Cases

### Quick Technology Check
Perform a fast analysis to get immediate results for a website's technology stack.

### Comprehensive Deep Scan
Enable deep scanning to analyze multiple pages and get detailed insights about the entire website architecture.

### Automated Processing
Use webhook notifications to integrate website analysis into your automated workflows and pipelines.

## Response Types

The endpoint returns different response types depending on the analysis conditions:

### 1. Standard Analysis Response

Returned when analysis completes within the `maxWait` time:

```json
{
  "profile": {
    "url": "https://example.com",
    "domain": "example.com",
    "scannedAt": "2024-01-15T10:30:00.000Z",
    "scan": {
      "type": "quick",
      "coverage": 1
    }
  },
  "stack": {
    "components": [...],
    "summary": {
      "total": 5,
      "byType": {
        "js-framework": 1,
        "hosting": 1
      }
    }
  },
  "identity": {
    "name": "Example Company",
    "domain": "example.com",
    "branding": {...},
    "social": {...}
  }
}
```

### 2. Cached Response

Returned when valid cached data is available:

```json
{
  "profile": {...},
  "stack": {...},
  "identity": {...},
  "cache": {
    "hit": true,
    "age": 5400,
    "taskId": "task_abc123"
  }
}
```

### 3. Async Scan Response

Returned when analysis exceeds `maxWait` and continues asynchronously:

```json
{
  "url": "https://example.com",
  "status": "scanning",
  "taskId": "task_xyz789",
  "estimatedDuration": "2-3 minutes",
  "statusUrl": "https://api.stacksee.com/api/v1/tasks/task_xyz789/status",
  "webhookConfigured": true,
  "message": "Analysis is taking longer than expected. Results will be sent to your webhook when complete."
}
```

## Async Processing

When analysis takes longer than `maxWait` seconds, the endpoint returns an async response with a `taskId`. You can:

1. **Poll for Results**: Use the `statusUrl` to check task status
2. **Webhook Notification**: Provide a `webhookUrl` to receive results automatically when complete

## Caching Behavior

The endpoint supports intelligent caching controlled by the `freshness` parameter:

- **`auto`** (default): Uses cached data if available and not stale, otherwise fetches fresh data
- **`cached`**: Always returns cached data if available, fails if no cache exists
- **`fresh`**: Always performs a fresh analysis, updates the cache

The `maxCacheAge` parameter controls cache staleness (default: 168 hours / 7 days).
